# 핵심 UX 플로우 (UX Flow Main)

**버전:** v0.1
**최종 수정일:** 2025-11-05

---

## 개요

이 문서는 BestPractice HR SaaS의 **핵심 유저 여정(User Journey)**을 정의한다.
각 유저 타입별로 가장 중요한 플로우를 단계별로 서술하며,
이는 `docs/business/business-plan.md`의 타겟 유저, 마케팅 메시지, 핵심 기능을 기반으로 한다.

**목표:**
- 유저가 서비스에서 **가치를 체감하는 순간**까지의 여정을 명확히 한다.
- 각 플로우가 **왜 중요한지** 한 줄로 설명해 개발 우선순위를 명확히 한다.

---

## 1. 점주(Store Owner) 플로우

### 1.1 첫 회원가입 → 매장 등록 → 온보딩 → 대시보드 확인

**왜 중요한가:**
점주가 **5분 안에 매장을 등록하고, 첫 매뉴얼을 만들어 직원에게 공유**하는 순간,
이 서비스가 "성공 매장의 운영 공식을 복제"한다는 핵심 가치를 체감한다.

**플로우:**

1. **랜딩 페이지 진입**
   - "성공하는 사장님의 비법을, 그대로 시스템으로" 메시지 노출
   - "무료로 시작하기" CTA 클릭

2. **회원가입 (이메일 + 비밀번호)**
   - 이메일 인증 링크 발송 (선택적)
   - 기본 정보 입력: 이름, 전화번호

3. **매장 정보 등록**
   - 매장명, 업종, 주소 입력
   - "내 매장은 카페입니다" → 시스템이 카페 업종 맞춤 템플릿 자동 추천

4. **온보딩: 5분 매뉴얼 만들기**
   - "첫 매뉴얼을 만들어보세요" 가이드 표시
   - 템플릿 선택: "오픈 준비 체크리스트" (카페 맞춤)
   - 자동 생성된 체크리스트 확인 (예: 에스프레소 머신 켜기, 재고 확인 등)
   - "직원 초대하기" 버튼 → 직원 이메일/전화번호 입력

5. **대시보드 첫 화면**
   - "오늘의 근무 현황" (아직 직원 없음 표시)
   - "이번 주 인건비" (₩0 표시)
   - "다음 할 일: 직원 초대 완료하기" 제안

**핵심 체감 순간:**
Step 4에서 **"내 매장 맞춤 매뉴얼이 5분 만에 자동 생성"**되는 경험.

---

### 1.2 직원 초대 → 교육 배정 → 출퇴근 확인 → ROI 체감

**왜 중요한가:**
점주가 **"직원 관리가 자동화되고, 인건비가 가시화"**되는 순간,
Free → Subscription 전환의 동기가 생긴다.

**플로우:**

1. **직원 초대**
   - 대시보드에서 "직원 추가" 클릭
   - 이메일/전화번호 입력 → 초대 링크 발송
   - 직원이 초대 수락 → 자동으로 "신규 직원 온보딩 매뉴얼" 배정

2. **교육 배정 자동화**
   - 시스템이 해당 직원에게 "오픈 준비", "음료 제조" 매뉴얼 자동 배정
   - 점주는 "교육 진행률" 대시보드에서 실시간 확인

3. **출퇴근 기록 시작**
   - 직원이 앱/웹에서 "출근" 버튼 클릭
   - 점주 대시보드에 "오늘 근무 중인 직원" 실시간 표시

4. **주간 리포트 확인 (ROI 체감)**
   - 주말에 "이번 주 인건비 리포트" 자동 발송
   - "예상 인건비: ₩850,000 / 실제 인건비: ₩780,000 (9% 절감)" 표시
   - "AI 스케줄링으로 전환하면 15% 더 절감 가능" 제안 → Subscription 전환 유도

**핵심 체감 순간:**
Step 4에서 **"인건비가 자동으로 계산되고, 절감 가능성"**이 보이는 순간.

---

### 1.3 Subscription 전환 → AI 스케줄링 사용 → Best Practice 모델 적용

**왜 중요한가:**
점주가 **"성공 매장의 스케줄링 패턴을 복제"**하는 경험을 통해,
고가 플랜(Best Practice)으로 업그레이드할 동기를 얻는다.

**플로우:**

1. **Subscription 플랜 선택**
   - "AI 법규 준수 + 스케줄링" 기능 설명 페이지
   - "Standard 플랜 시작하기" (₩29,000/월) 클릭
   - 결제 정보 입력 (카드/계좌)

2. **AI 스케줄링 첫 사용**
   - "다음 주 스케줄 자동 생성" 클릭
   - 시스템이 과거 매출 데이터, 요일별 방문객 패턴 분석
   - "월요일 오전 2명, 오후 3명" 추천 스케줄 자동 생성
   - 점주가 미세 조정 후 저장

3. **Best Practice 모델 템플릿 추천**
   - 대시보드에 "성공하는 카페 상위 10%의 스케줄 패턴" 배너 표시
   - "Best Practice 플랜 무료 체험 (14일)" 제안
   - 클릭 시 "성공 매장 모델: 강남 OO카페 스케줄 복제하기" 화면 표시

4. **벤치마킹 리포트 확인**
   - "우리 매장 vs 성공 매장 비교" 리포트
   - "성공 매장은 주말 오전에 1명 더 배치해 대기 시간 30% 단축" 인사이트 제공
   - "이 패턴 적용하기" 클릭 → 자동으로 다음 주 스케줄 반영

**핵심 체감 순간:**
Step 4에서 **"성공 매장의 운영 공식을 클릭 한 번에 복제"**하는 경험.

## 2. 알바생(Part-timer/Employee) 플로우

### 2.1 초대 수락 → 계정 생성 → 교육 매뉴얼 학습 → 출퇴근 기록

**왜 중요한가:**
알바생이 **"성장하는 경력 관리 플랫폼"**으로 인식하는 순간,
재고용 확률과 매칭 풀 품질이 향상된다.

**플로우:**

1. **초대 링크 수신**
   - 점주가 보낸 초대 링크 클릭 (SMS/이메일)
   - "OO카페에서 당신을 초대했습니다" 화면

2. **계정 생성**
   - 이메일/비밀번호 설정 (소셜 로그인 옵션)
   - 기본 정보 입력: 이름, 전화번호, 근무 가능 시간대

3. **온보딩: 교육 매뉴얼 자동 배정**
   - "OO카페 신규 직원 온보딩" 매뉴얼 자동 표시
   - 체크리스트: "에스프레소 머신 사용법 영상 보기", "POS 시스템 테스트" 등
   - 각 항목 완료 시 "진행률 +20%" 표시

4. **첫 출근: 출퇴근 기록**
   - 앱에서 "출근" 버튼 클릭 (GPS/QR 코드 인증)
   - "오늘 배정된 업무: 오픈 준비 체크리스트" 자동 표시
   - 체크리스트 완료 시 "스킬 획득: 오픈 준비 마스터" 배지 부여

5. **주간 성장 리포트**
   - "이번 주 완료한 교육: 5개, 획득 스킬: 음료 제조 Lv.2" 표시
   - "다음 목표: 바리스타 인증 과정 (3주 완료)" 제안

**핵심 체감 순간:**
Step 4~5에서 **"단순 알바가 아니라, 성장하는 경력"**을 느끼는 순간.

---

### 2.2 스킬 배지 획득 → 경력 리포트 확인 → 재고용/매칭 기회

**왜 중요한가:**
알바생이 **"검증된 인재 풀에 편입"**되면서,
매칭 레이어의 네트워크 효과가 발동한다.

**플로우:**

1. **스킬 배지 획득**
   - 매뉴얼 10개 완료 시 "음료 제조 마스터" 배지 획득
   - 점주 평가 4.5점 이상 유지 시 "신뢰 직원" 배지 획득

2. **경력 리포트 생성**
   - "내 경력 보기" 클릭 → PDF 다운로드 가능
   - "OO카페 근무 6개월, 획득 스킬: 음료 제조 Lv.3, POS 관리 Lv.2"
   - "신뢰도: 4.8/5.0 (20개 평가 기반)"

3. **재고용 제안 수신**
   - 다른 카페 점주가 "음료 제조 마스터 배지 보유 직원" 검색
   - 알바생에게 "근처 △△카페에서 당신을 원합니다" 푸시 알림
   - 클릭 시 매장 정보, 시급, 근무 조건 확인 → 수락/거절

4. **AI 경력 코칭**
   - "바리스타 자격증을 취득하면 시급 +15% 매장 매칭 가능" 제안
   - "추천 교육 과정: OO아카데미 (파트너사 할인 50%)" 링크 제공

**핵심 체감 순간:**
Step 3에서 **"검증된 경력으로 더 좋은 기회"**를 받는 순간.

## 3. 전문가 파트너(Professional Partner) 플로우

### 3.1 파트너 등록 → 점주 계정 연결 → 리포트/자문 화면 진입

**왜 중요한가:**
노무사/세무사가 **"고객 유입 채널 + 지속 수익"**을 얻는 순간,
3자 확산 네트워크의 핵심 축이 작동한다.

**플로우:**

1. **파트너 프로그램 신청**
   - "전문가 파트너 되기" 페이지 (노무사/세무사 대상)
   - 사업자 정보, 자격증 업로드
   - 승인 후 "파트너 대시보드" 접근 권한 부여

2. **고객(점주) 연결**
   - 파트너가 기존 고객에게 "BestPractice HR 무료 체험" 링크 공유
   - 점주가 링크로 가입 시 자동으로 "소속 파트너: OO 노무사" 태그
   - 파트너 대시보드에 "연결된 점주: 5명" 표시

3. **고객 리포트 자동 생성**
   - 파트너 대시보드에서 "고객 법규 준수 현황" 확인
   - "경고: △△카페, 주 52시간 초과 위험" 알림
   - "고객에게 자문 메시지 보내기" 클릭 → 점주에게 자동 발송

4. **수익 쉐어 확인**
   - 연결된 점주가 Subscription 전환 시 "구독 수익 10% 쉐어" 자동 적립
   - "이번 달 수익: ₩150,000 (5명 고객 구독료의 10%)" 표시
   - 출금 신청 가능

**핵심 체감 순간:**
Step 4에서 **"단발 자문이 아닌, 지속 수익 채널"**을 체감하는 순간.

---

### 3.2 AI 자문 도구 활용 → 프리미엄 고객 전환

**왜 중요한가:**
파트너가 **"AI 기반 자문 도구"**를 활용하면,
고객당 케어 시간이 줄어들고, 더 많은 점주를 관리할 수 있다.

**플로우:**

1. **AI 법규 준수 리포트 활용**
   - "이번 주 법규 위험 고객" 자동 알림
   - 클릭 시 "근로기준법 위반 가능성 3건" 상세 리포트
   - "자동 해결 제안서 생성" → 점주에게 1클릭 발송

2. **고객 업그레이드 제안**
   - "△△카페는 Best Practice 플랜 적합 고객" AI 추천
   - 파트너가 "업그레이드 제안 메시지" 전송
   - 고객 전환 시 수익 쉐어 +5% 추가

3. **월간 파트너 리포트**
   - "관리 고객: 20명, 이번 달 신규 유입: 3명"
   - "평균 법규 준수율: 92% (업계 평균 78%)"
   - "추천 고객 확대 전략: OO 협회 세미나 발표 제안"

**핵심 체감 순간:**
Step 1~2에서 **"AI가 자문 업무를 자동화"**하는 경험.

## 4. 공통 크로스 플로우

### 4.1 알림 및 푸시 전략

**목적:**
유저가 **"플랫폼에 돌아올 이유"**를 지속적으로 제공.

**주요 알림 시나리오:**

1. **점주:**
   - "오늘 출근 예정 직원: 3명, 스케줄 확인하기"
   - "이번 주 인건비 ₩780,000 (예산 대비 9% 절감)"
   - "AI 추천: 주말 오전 1명 더 배치하면 매출 +12%"

2. **알바생:**
   - "내일 근무 시작 2시간 전" 알림
   - "새로운 교육 매뉴얼 배정: 고객 응대 스킬"
   - "스킬 레벨업! 음료 제조 Lv.3 달성"

3. **파트너:**
   - "경고: △△카페 법규 위험, 자문 필요"
   - "신규 고객 유입: OO식당이 당신을 파트너로 선택"
   - "이번 달 수익 ₩200,000 달성, 출금 가능"

---

### 4.2 재방문 유도 플로우

**목적:**
Churn 방지 및 재활성화.

**주요 시나리오:**

1. **7일 미접속 점주:**
   - "OO님, 직원 근태 기록이 쌓이고 있어요. 리포트 확인하기"
   - 클릭 시 "이번 주 자동 생성된 인건비 리포트" 표시

2. **30일 미접속 알바생:**
   - "OO님, 새로운 매칭 기회가 있어요. 근처 3개 매장 확인하기"
   - 클릭 시 "검증된 매장 리스트" 표시

3. **Subscription 해지 고객:**
   - "해지 후 30일 이내 재구독 시 첫 달 50% 할인"
   - "Best Practice 무료 체험 2주 제공"

## 5. 플로우 우선순위 (MVP Phase 1)

Phase 1 개발에서는 다음 플로우를 우선 구현한다:

| 순위 | 플로우 | 이유 |
|------|--------|------|
| 1 | 점주: 회원가입 → 매장 등록 → 5분 매뉴얼 생성 | 핵심 가치 체감 |
| 2 | 점주: 직원 초대 → 교육 배정 → 출퇴근 확인 | Free → Subscription 전환 동기 |
| 3 | 알바생: 초대 수락 → 교육 학습 → 출퇴근 기록 | 양면 네트워크 활성화 |
| 4 | 점주: ROI 리포트 확인 → Subscription 전환 | BEP 달성 핵심 |

**Phase 2 이후:**
- 점주: Best Practice 모델 적용
- 알바생: 스킬 배지 → 재고용 매칭
- 파트너: 고객 연결 → 수익 쉐어

## 6. 핵심 UX 원칙

모든 플로우 설계 시 다음 원칙을 준수한다:

1. **5분 내 가치 체감**
   - 유저가 첫 로그인 후 5분 안에 핵심 가치를 경험해야 한다.

2. **자동화 우선**
   - 유저가 입력할 데이터를 최소화하고, AI/템플릿이 대신 채운다.

3. **성장 가시화**
   - 점주는 "인건비 절감", 알바생은 "스킬 레벨", 파트너는 "수익"을 숫자로 본다.

4. **네트워크 효과 유도**
   - 점주→알바생 초대, 파트너→점주 유입 등 **"초대가 자연스럽게 발생"**하는 구조.

5. **데이터 플라이휠**
   - 사용할수록 AI 추천이 정확해지고, 더 나은 Best Practice 모델이 제공된다.
