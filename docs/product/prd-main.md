**제품명:** BestPractice HR SaaS

## 1. 제품 개요 (Product Overview)

### 1.1 제품 비전
성공하는 매장의 HR·운영 베스트 프랙티스를 데이터 기반으로 모델링하여,
누구나 따라 하면 성과가 나는 자영업 HR SaaS 플랫폼

### 1.2 목표 (Goals)
- 점주: 채용 실수 최소화, 인건비 최적화, 교육 시간 70% 절감
- 알바생: 스킬 성장, 검증된 경력 증명
- 플랫폼: 3년 내 10,000개 매장 확보, HR SaaS 시장 표준화

### 1.3 성공 지표 (Success Metrics)
- 매장당 평균 인건비 절감률: 10% 이상
- 직원 교육 시간 단축: 70% 이상
- 알바생 재고용률: 40% 이상
- NPS (Net Promoter Score): 50+ (3년 목표)
- ARPU (Average Revenue Per User): ₩85,000/월

## 3. 문제 정의 (Problem Statement)

### 3.1 점주의 문제
**P1. 채용 실수로 인한 손실**
- 적합하지 않은 직원 채용 → 재교육/재채용 비용 발생
- 현재: 주먹구구식 면접, 직감 의존

**P2. 과다한 인건비**
- 수요 예측 없는 스케줄링 → 불필요한 인건비 지출
- 현재: 엑셀/수기 관리, 비효율적 배치

**P3. 직원 교육 부담**
- 신규 직원마다 반복적인 OJT
- 현재: 매뉴얼 부재, 구두 전달, 일관성 없는 교육

**P4. 법규 리스크**
- 근로기준법, 최저임금, 주휴수당 등 복잡한 법규
- 현재: 법규 위반 시 벌금/분쟁 발생

### 3.2 직원의 문제
**P5. 불명확한 업무 기준**
- 매장마다 다른 기준, 주먹구구식 지시
- 현재: 혼란, 스트레스, 실수 증가

**P6. 경력 증명 어려움**
- 단순 알바 경력 → 이력서에 의미 없음
- 현재: 성장 기회 부재, 동기부여 낮음

### 3.3 파트너의 문제
**P7. 단발성 컨설팅 한계**
- 노무사/세무사: 1회성 자문 → 지속 수익 어려움
- 현재: 반복 고객 확보 어려움

## 4. 솔루션 (Solution)

### 4.1 핵심 컨셉
**"성공 매장의 HR 베스트 프랙티스를 AI로 학습하여 자동 복제하는 SaaS"**

### 4.2 5-Layer 제품 구조

| 레이어 | 핵심 기능 | 타겟 유저 |
|--------|----------|----------|
| **① Free Layer** | 5분 매뉴얼, 기본 근태, 교육 자동화 | 모든 점주 |
| **② Subscription Layer** | AI 법규 준수, AI 스케줄링, 효율 분석 | 성장 지향 점주 |
| **③ Best Practice Layer** | 성공 매장 모델 템플릿, 실시간 벤치마킹 | 프리미엄 점주 |
| **④ Matching Layer** | 검증 인재 추천, AI 경력 코칭 | 모든 점주·직원 |
| **⑤ Data Layer** | 업종별 HR 지표 리포트, API | 파트너·대기업 |

## 5. 기능 우선순위 (Feature Priority)

### 5.1 MUST-HAVE (v1.0 출시 필수)

**M1. 회원가입 & 인증**
- 점주: 이메일/소셜 로그인
- 직원: 초대 링크 기반 가입
- 역할 기반 접근 제어 (RBAC)

**M2. 5분 매뉴얼 (Free Layer)**
- 직무별 업무 매뉴얼 생성·공유
- 체크리스트 기반 학습 인증
- 모바일 접근 가능

**M3. 기본 근태 관리**
- 출퇴근 기록 (GPS/QR 코드)
- 근무 시간 자동 계산
- 주휴수당 자동 계산

**M4. 스케줄 관리**
- 주간/월간 스케줄 생성
- 직원별 근무 시간 배정
- 변경 사항 알림

**M5. 급여 계산 (기본)**
- 시급 × 근무 시간 자동 계산
- 주휴수당 포함
- 급여 명세서 생성

**M6. 직원 프로필 & 스킬 배지**
- 직원 기본 정보 관리
- 완료한 교육/매뉴얼 기반 스킬 배지 부여
- 경력 리포트 생성

---

### 5.2 SHOULD-HAVE (v1.1~v2.0)

**S1. AI 수요 예측 스케줄링 (Subscription Layer)**
- 과거 매출·방문객 데이터 기반 수요 예측
- 최적 인력 배치 자동 추천
- 인건비 효율 시뮬레이션

**S2. AI 법규 준수 체크**
- 근로기준법 위반 사항 자동 감지
- 주휴수당, 연차, 최저임금 준수 알림
- 법규 변경 사항 자동 업데이트

**S3. 인건비 효율 대시보드**
- 매출 대비 인건비 비율 (Labor Cost %)
- 업종별 벤치마크 비교
- 효율 개선 제안

**S4. 채용 공고 & AI 적합도 테스트**
- 성공 매장 공고 문구 템플릿
- 지원자 직무 적합도 자동 평가
- 면접 질문 추천

**S5. 검증 인재 매칭 (Matching Layer)**
- 스킬 배지 기반 A급 인재 추천
- 재고용 인재 Pool
- 매칭 성공 시 수수료 모델

**S6. 성과 평가 & 피드백 자동화**
- 주간/월간 자동 피드백 템플릿
- 직원별 성과 추적
- 성장 리포트

---

### 5.3 WON'T-HAVE (현 단계에서 제외)

**W1. 급여 자동 이체**
- 이유: 금융 라이선스 필요, 초기에는 제외
- 대안: 급여 명세서 생성 후 수동 이체 안내

**W2. 화상 면접 기능**
- 이유: 복잡도 높음, 외부 툴 연동 우선
- 대안: Zoom/Meet 링크 공유

**W3. 재고 관리**
- 이유: HR 영역 벗어남, POS 연동으로 대체
- 대안: POS API 연동

**W4. 직원용 커뮤니티 (SNS)**
- 이유: 운영 부담, 초기 단계에 불필요
- 대안: 알림·피드백 기능으로 대체

## 6. 주요 유저 스토리 & Acceptance Criteria

### 6.1 점주 (Store Owner)

**US-01: 매뉴얼 생성**
- **As a** 점주
- **I want to** 직무별 5분 매뉴얼을 간단히 생성하고
- **So that** 신규 직원 교육 시간을 단축할 수 있다

**Acceptance Criteria:**
- AC-01: 점주가 "매뉴얼 생성" 버튼을 클릭하면 템플릿 선택 화면이 나타난다
- AC-02: 업종별 템플릿 (카페, 음식점, 소매점 등) 중 하나를 선택할 수 있다
- AC-03: 매뉴얼에 체크리스트 항목을 추가/수정/삭제할 수 있다
- AC-04: 완성된 매뉴얼을 저장하면 직원에게 자동 공유된다
- AC-05: 매뉴얼은 모바일에서도 동일하게 보인다

---

**US-02: 스케줄 생성**
- **As a** 점주
- **I want to** 주간 근무 스케줄을 생성하고
- **So that** 직원별 근무 시간을 효율적으로 배정할 수 있다

**Acceptance Criteria:**
- AC-06: 점주가 "스케줄 생성" 화면에서 주간/월간 뷰를 선택할 수 있다
- AC-07: 드래그앤드롭으로 직원을 시간대에 배정할 수 있다
- AC-08: 직원별 주간 근무 시간 합계가 자동 계산된다
- AC-09: 법정 근로시간 초과 시 경고 메시지가 나타난다
- AC-10: 스케줄 저장 시 직원에게 푸시 알림이 전송된다

---

**US-03: 급여 계산**
- **As a** 점주
- **I want to** 직원별 급여를 자동 계산하고
- **So that** 실수 없이 정확한 급여를 지급할 수 있다

**Acceptance Criteria:**
- AC-11: 월말에 "급여 계산" 버튼을 클릭하면 모든 직원의 급여가 자동 계산된다
- AC-12: 시급 × 근무 시간 + 주휴수당이 포함된다
- AC-13: 직원별 급여 명세서를 PDF로 다운로드할 수 있다
- AC-14: 급여 명세서에는 근무 일수, 시간, 시급, 주휴수당이 명시된다
- AC-15: 최저임금 미달 시 경고 메시지가 나타난다

---

**US-04: AI 스케줄링 추천 (Subscription)**
- **As a** 점주 (Premium 구독자)
- **I want to** 과거 매출 데이터 기반으로 최적 스케줄 추천을 받고
- **So that** 인건비를 절감하면서 서비스 품질을 유지할 수 있다

**Acceptance Criteria:**
- AC-16: Premium 플랜 구독자는 "AI 추천" 버튼이 활성화된다
- AC-17: 과거 4주 매출·방문객 데이터를 분석하여 시간대별 필요 인력을 추천한다
- AC-18: 추천 스케줄과 현재 스케줄의 인건비 차이를 시뮬레이션으로 보여준다
- AC-19: 추천 스케줄을 원클릭으로 적용할 수 있다
- AC-20: AI 추천 정확도를 주간 단위로 리포트한다

---

### 6.2 직원 (Part-timer)

**US-05: 매뉴얼 학습**
- **As a** 신규 직원
- **I want to** 5분 매뉴얼을 보고 체크리스트를 완료하고
- **So that** 빠르게 업무를 익힐 수 있다

**Acceptance Criteria:**
- AC-21: 직원은 모바일 앱에서 배정된 매뉴얼 목록을 볼 수 있다
- AC-22: 매뉴얼을 열면 체크리스트 항목들이 나타난다
- AC-23: 각 항목을 완료하면 체크 표시가 된다
- AC-24: 모든 항목 완료 시 스킬 배지가 자동 부여된다
- AC-25: 완료 상태는 점주 대시보드에 실시간 반영된다

---

**US-06: 출퇴근 기록**
- **As a** 직원
- **I want to** 모바일 앱으로 간단히 출퇴근을 기록하고
- **So that** 정확한 근무 시간을 자동으로 집계할 수 있다

**Acceptance Criteria:**
- AC-26: 직원은 앱에서 "출근" 버튼을 누르면 GPS/QR 코드 인증 옵션이 나타난다
- AC-27: GPS 반경 50m 이내에서만 출근 가능 (점주 설정)
- AC-28: QR 코드 스캔 시 출근 기록이 즉시 저장된다
- AC-29: "퇴근" 버튼 클릭 시 근무 시간이 자동 계산되어 표시된다
- AC-30: 출퇴근 기록 수정 요청 시 점주 승인이 필요하다

---

**US-07: 경력 리포트 확인**
- **As a** 직원
- **I want to** 내가 획득한 스킬 배지와 경력 리포트를 확인하고
- **So that** 다음 취업 시 검증된 경력으로 활용할 수 있다

**Acceptance Criteria:**
- AC-31: 직원은 "내 프로필" 화면에서 스킬 배지 목록을 볼 수 있다
- AC-32: 스킬 배지는 아이콘 + 이름 + 획득 날짜로 표시된다
- AC-33: "경력 리포트 다운로드" 버튼을 클릭하면 PDF가 생성된다
- AC-34: 경력 리포트에는 근무 기간, 총 근무 시간, 획득 스킬이 포함된다
- AC-35: 경력 리포트는 QR 코드로 검증 가능하다 (향후 구현)

---

### 6.3 파트너 (Professional)

**US-08: 고객 매장 현황 모니터링**
- **As a** 노무사 (파트너)
- **I want to** 내 고객 매장들의 법규 준수 상태를 한눈에 보고
- **So that** 사전에 리스크를 방지하고 컨설팅할 수 있다

**Acceptance Criteria:**
- AC-36: 파트너 대시보드에서 고객 매장 목록을 볼 수 있다
- AC-37: 각 매장의 법규 준수 점수 (0-100)가 표시된다
- AC-38: 위반 사항이 있는 매장은 빨간색으로 강조된다
- AC-39: 매장 클릭 시 상세 리포트 (위반 항목, 개선 제안)를 볼 수 있다
- AC-40: 주간 리포트를 이메일로 자동 수신할 수 있다

## 7. 비기능 요구사항 (Non-Functional Requirements)

### 7.1 성능 (Performance)
- **NFR-01:** 주요 API 응답 시간 p95 < 500ms
- **NFR-02:** 웹 최초 로드 시간 (FCP) < 2초
- **NFR-03:** 모바일 앱 최초 로드 시간 < 3초
- **NFR-04:** 동시 접속자 1,000명 지원 (초기), 10,000명 지원 (3년)

### 7.2 보안 (Security)
- **NFR-05:** HTTPS 필수, TLS 1.2 이상
- **NFR-06:** 비밀번호 bcrypt 해싱 (cost factor 12)
- **NFR-07:** JWT 기반 인증, 토큰 만료 시간 24시간
- **NFR-08:** RBAC (Role-Based Access Control) 구현
- **NFR-09:** 민감 정보 (주민번호, 계좌번호) 암호화 저장
- **NFR-10:** GDPR/PIPA 준수 (개인정보 삭제 요청 처리)

### 7.3 가용성 (Availability)
- **NFR-11:** SLA 99.5% uptime (초기), 99.9% (3년)
- **NFR-12:** DB 자동 백업 (일 1회, 보관 30일)
- **NFR-13:** 장애 복구 시간 (RTO) < 4시간
- **NFR-14:** 데이터 손실 허용 범위 (RPO) < 1시간

### 7.4 확장성 (Scalability)
- **NFR-15:** 수평 확장 가능한 API 서버 아키텍처
- **NFR-16:** DB Read Replica 구성 (읽기 부하 분산)
- **NFR-17:** Redis 캐싱으로 반복 쿼리 최적화

### 7.5 사용성 (Usability)
- **NFR-18:** 모바일 반응형 디자인 (iOS/Android/Web)
- **NFR-19:** 접근성 (WCAG 2.1 AA 수준)
- **NFR-20:** 다국어 지원 준비 (i18n 구조, 초기 한국어만)
